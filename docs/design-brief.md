# 그늘 (Geuneul) — 디자인·API 계약 레퍼런스 (구현 완료·라이브)

> 아래 디자인 토큰·화면·API 계약으로 프론트엔드 4화면이 구현·배포됐다(https://geuneul.vercel.app). API 계약은 라이브 검증된 실제 값이며, 이 문서는 그 참조 스냅샷으로 남긴다.
> 백엔드: `http://geuneul-alb-1266310270.ap-northeast-2.elb.amazonaws.com` (Swagger `/swagger-ui.html`)

## 1. 제품 한 줄

**여름 생존 지도.** 폭염·장마철에 "지금 쉬어갈 곳"(무더위쉼터·화장실·음수대·도서관·지하상가)을
현재 위치 기준으로 찾아주는 모바일 웹(PWA). 태그라인: **"지금 쉬어갈 그늘을 찾아드립니다."**

**톤:** 한여름 뙤약볕에서 급하게 꺼내 쓰는 앱 — 시원함·그늘·물을 연상시키는 차분한 색(진초록/청록/화이트),
큰 터치 타깃, 한 손 조작. 공포 조장 금지(침수 경고도 "우회 권장" 수준의 차분한 표현).

## 2. 화면 (MVP 4개)

### ① 홈 지도 (메인)
- **Kakao Maps 전면 배치** + 현재 위치 버튼.
- 상단: 카테고리 필터 칩(가로 스크롤) — 아래 8종 참고. 다중 선택.
- 마커: 카테고리별 아이콘. **마커 색상은 카테고리가 아니라 "지금 갈만함 점수" 구간으로 칠할 예정**
  (P3에서 추가: 초록=지금 좋음/노랑=보통/**회색=정보 부족**). 지금은 전부 회색 상태로 디자인하되
  3색 변형을 마커 컴포넌트에 예약해 둘 것.
- 하단 시트(바텀시트): 반경 검색 결과 리스트(이름·카테고리 라벨·거리 m·주소). 지도 마커와 연동.
- 빈 상태: "이 근처엔 아직 정보가 없어요" + 반경 넓히기 버튼.

### ② 장소 상세 (바텀시트 확장 or 페이지)
- 이름, 카테고리 라벨, 주소, 거리, 지도 미니뷰.
- (P2 예약 영역) 최근 제보 타임라인("10분 전 · 지금 시원해요"), 후기 목록(별점·코멘트), 사진.
- 액션: 길찾기(카카오맵 열기), 공유, (P2) 제보하기·후기쓰기.

### ③ "지금 급해요" 추천 탭
- 시나리오 버튼 3개: **"화장실 급함"** / "잠깐 쉬어갈 곳(30분)" / "비 피할 곳".
- 탭하면 kNN 최근접 결과를 거리순 카드 리스트로(1위는 크게, 도보 예상시간 = 거리m ÷ 67).
- 이 화면이 그늘의 정체성 — 지도 탐색이 아니라 **즉답**.

### ④ 제보하기 (P2 예약 — 레이아웃만)
- 한 탭 상태 제보: 이모지 그리드(자리있어요🪑/붐벼요👥/시원해요🧊/더워요🥵/벌레많아요🐛/화장실깨끗🚻/침수됐어요🌊 등).
- 사진 1장 + 한 줄 코멘트(선택). 로그인 없이도 익명 제보 가능(로그인 시 신뢰도 반영 배지).

## 3. API 계약 (확정 · 라이브 검증됨)

```
GET /places?lat={위도}&lng={경도}&radius={m, 기본800, 최대5000}&category={카테고리}&limit={기본100}
GET /places?bounds=west,south,east,north&category=...        # 지도 뷰포트
GET /places/nearest?lat=&lng=&category=&limit={기본5, 최대50} # 최근접 kNN
GET /places/{id}
```

**응답 예시 (프로덕션 실측):**
```json
[{
  "id": 1,
  "name": "숭실대학교 중앙도서관",
  "category": "LIBRARY",
  "categoryLabel": "도서관",
  "address": "서울 동작구 상도로 369",
  "lat": 37.4962,
  "lng": 126.9573,
  "source": "seed",
  "distanceM": 20.9
}]
```
- `distanceM`은 반경/최근접 검색에서만 존재(bounds/단건엔 null) — 리스트 카드에 "21m" 형태로 표기.
- 에러(400 등)는 Spring 기본 오류 JSON: `{"timestamp":"...","status":400,"error":"Bad Request","path":"/places"}` — 바디의 `status`/`error`로 표시. (RFC7807 ProblemDetail 전환은 백엔드 후속 옵션.)

## 5. 나중에 들어올 것 (자리만 예약, 지금 픽셀 만들지 말 것)

- **survival_score 3색 마커** + 점수 배지(P3) — 마커·카드에 색/배지 슬롯만.
- 제보 freshness 표시("10분 전 제보") — 상세·카드에 타임스탬프 슬롯.
- 로그인/프로필(카카오/구글), 신뢰도 배지(P2).
- AI 한줄 요약(상세 상단 인용구 스타일, P3).

## 6. 제약·비목표

- **모바일 웹(PWA) 우선.** 뷰포트 375~430px 기준, 데스크톱은 확장 대응 정도.
- 지도는 Kakao Maps JS SDK(마커·클러스터러 사용 전제).
- **친구/팔로우/스토리 없음** — 공개 정보 서비스. 커뮤니티 요소는 장소 중심(제보/후기)뿐.
- 다크모드는 2차(폭염 앱 특성상 야외 시인성 = 라이트 우선).
