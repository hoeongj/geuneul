# ADR-0023. 커먼스 세이프 팔로우 — 작성자 공개 프로필 + "수만 공개, 목록은 비공개" 팔로우

- 상태: 승인(구현 반영, 2026-07-11)
- 관련: `domain.follow`(신규), `V17__follows.sql`, `domain.activity`(N6 후기 쿼리 재사용),
  `ReviewResponse.authorId`(신규 필드), 프론트 `UserProfileOverlay`·`FollowingSection`·`selectedUser` 컨텍스트,
  SPEC.md §0-9(그늘은 공개 커먼스, 친구망 아님)·§1(신뢰도)·§0-2(범위 절제)
- 후속: BACKLOG N7

## 문제(Context)

실사용 피드백: "특정 사람 후기가 좋아서 가끔 찾아보고 싶다." 즉 **좋은 작성자를 다시 찾는** 욕구다. 자연스러운 해법은
팔로우지만, 그늘의 정체성은 **공개 커먼스지 폐쇄 친구망이 아니다**(SPEC.md §0-9 — 맛핀에서 넘어온 것 중
친구/그룹 그래프·스토리·팔로우 피드는 **의도적으로 제외**). 풀 소셜 그래프(팔로워 목록·피드·맞팔 알림)를 넣으면
간판(지리공간 + UGC 스코어링)이 흐려지고 앱이 "또 하나의 SNS"가 된다. 그렇다고 팔로우를 아예 안 하면 "다시 찾기"
욕구를 못 채운다.

## 결정(Decision)

**"커먼스 세이프" 팔로우** — 팔로우를 소셜 그래프가 아니라 **① 사적 북마크 + ② 신뢰/인기 신호**로만 쓴다.

### 1) 작성자 공개 프로필 `GET /users/{id}` (공개)

닉네임 · **신뢰도(trust_score 배지)** · **팔로워 "수"(공개 카운트)** · **그 사람의 공개 후기 목록**. 팔로우 안 해도
커먼스라 **누구나 본다**. 후기 목록은 N6 "내 글 관리"의 `ReviewRepository.findByUserIdWithPlace`를 **그대로 재사용**
(작성자별 후기 조회는 동일 로직 — BACKLOG 지시). 로그인 뷰어면 `following`(내가 팔로우 중인지)을 채워 버튼 상태를 준다.

### 2) 팔로우 `POST/DELETE /users/{id}/follow` (로그인, 멱등)

`follows(follower_id, followee_id, created_at)` + `UNIQUE(follower_id, followee_id)`(멱등) +
`CHECK(follower_id <> followee_id)`(자기 팔로우 금지). 동시 이중요청은 UNIQUE가 backstop. 응답 `{following, followerCount}`로
버튼·카운트 즉시 반영.

### 3) 내 팔로잉 `GET /me/following` — **나만** 본다

마이페이지에서 내가 팔로우한 작성자를 재방문하는 **사적 목록**. 이게 "좋은 작성자 다시 찾기"의 실제 도구다.

### 4) 하지 않는 것(§0-9 방어 — 명시적 비목표)

- **팔로워 "목록"은 어디에도 없다.** 프로필엔 **수(count)만** 노출한다(허영 리스트 배제). 팔로워를 조회하는 쿼리·엔드포인트
  자체를 만들지 않는다.
- **팔로우 피드 없음**(팔로우한 사람의 새 글을 모아 보여주는 타임라인 = SNS화 → 배제).
- **맞팔 역학·상호 팔로우 알림·스토리 없음.**

## 근거(Why this shape)

- **팔로워 "수"만 공개** = 신뢰/인기 신호(§1 신뢰도 축 강화 — 평판 시스템)로만 쓰고, 허영 지표(누가 나를
  팔로우하는지 리스트)·피드는 배제 → 커먼스 정체성 유지.
- **팔로잉은 사적 북마크** = 남에게 안 보이니 "관심 작성자 조용히 모으기"가 되고, 소셜 압박(맞팔 강요)이 없다.
- 검토 대안: (a) **프로필만, 팔로우 없음** — "다시 찾기"를 못 해결. (b) **풀 소셜 그래프** — §0-9 위배, 간판 흐림.
  (c) **비공개 관심작성자(북마크형)만** — 팔로워 수라는 신뢰 신호를 못 살림. → 사용자 확정: **(팔로워 수 공개 + 팔로잉 사적)**
  절충(2026-07-11).

## 결과(Consequences)

- 신뢰도/평판 축이 강화된다(작성자 프로필에 trust_score + 팔로워 수). 간판(지리·스코어링)은 그대로.
- `ReviewResponse`에 `authorId` 추가(닉네임 탭 → 프로필). 후기는 익명이 아니라 항상 작성자가 있어 노출이 안전(공개 프로필과 정합).
- 팔로워 목록 쿼리를 안 만든 건 **의도**다 — 나중에 "친구 추천/피드"를 넣고 싶어도 이 ADR이 그 경계를 명시한다(범위 방어).
- 스키마 V17 1개. 프론트는 오버레이(장소 상세와 동일 패턴) + 마이페이지 팔로잉 섹션.
